# SIPFLOW Docs

SIPFLOW 문서는 역할별로 분리한다. 현재 기준 source of truth는 아래 순서를 따른다.

## 빠른 이동

- 프로젝트 개요: [project/overview.md](project/overview.md)
- 현재 상태: [project/state.md](project/state.md)
- 제품 로드맵: [project/roadmap.md](project/roadmap.md)
- 활성 요구사항: [product-specs/active/v1.4-core-call-stability.md](product-specs/active/v1.4-core-call-stability.md)
- 설계/리서치: [design-docs/research/](design-docs/research/)
- 실행 계획: [PLANS.md](PLANS.md), [exec-plans/](exec-plans/)

## 문서 계층

- `project/`: 살아있는 프로젝트 설명, 현재 상태, 장기 로드맵
- `product-specs/`: 마일스톤 요구사항. `active/`는 현재 작업, `archive/`는 완료된 마일스톤
- `design-docs/`: 구현 전후 설계 근거와 리서치
- `exec-plans/`: 실행 가능한 plan 문서와 완료 이력
- `archive/`: 더 이상 source of truth가 아닌 레거시 산출물
