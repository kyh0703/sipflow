**Step 2: Create sipStore for call state tracking**

Create `frontend/src/stores/sipStore.ts` using Zustand with actions object pattern (established in 01-02):

```typescript
interface CallState {
  callID: string
  nodeID: string
  state: 'dialing' | 'ringing' | 'established' | 'terminated' | 'failed'
  statusCode?: number
  error?: string
}

interface SIPState {
  activeCalls: Map<string, CallState>  // callID -> state
  callHistory: CallState[]              // recent completed calls
}

interface SIPActions {
  updateCallState: (event: CallState) => void
  clearHistory: () => void
  initCallStateListener: () => void
}
```

- `initCallStateListener()`: Use Wails `EventsOn("sip:callState", callback)` to listen for call state events from backend. On each event, update activeCalls map. When state is "terminated" or "failed", move from activeCalls to callHistory.
- Call `initCallStateListener()` from the store creation (self-initializing on import).
- Use `EventsOff` pattern for cleanup if needed.

Import EventsOn from `@wailsapp/runtime` (existing pattern from project).
