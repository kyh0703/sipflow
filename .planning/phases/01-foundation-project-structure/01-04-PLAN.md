  <action>
    1. Create `frontend/src/services/flowService.ts`:
       - Import Wails auto-generated FlowService from `../../wailsjs/go/handler/FlowService`
       - Import `ApiResponse` type from `../types/responses`
       - Create typed wrapper functions that call the Wails bindings:

         ```typescript
         export async function listFlows(): Promise<ApiResponse<Flow[]>> {
           return await FlowService.ListFlows();
         }

         export async function createFlow(name: string, description: string): Promise<ApiResponse<Flow>> {
           return await FlowService.CreateFlow(name, description);
         }

         export async function getFlow(id: number): Promise<ApiResponse<Flow>> {
           return await FlowService.GetFlow(id);
         }

         export async function deleteFlow(id: number): Promise<ApiResponse<boolean>> {
           return await FlowService.DeleteFlow(id);
         }
         ```

       - Define a `Flow` interface that matches the Wails-generated type (or import it from wailsjs/go/models if generated)
       - The wrapper adds type safety and provides a single import point for flow operations

    2. Update `frontend/src/App.tsx`:
       - In the useEffect that runs `initializeEventHandshake()`, after handshake completes:
         a. Call `listFlows()` from flowService
         b. Log the result to console: `console.log('Flow list:', result)`
         c. If result.success, display the flow count in the main content area
       - This serves as a smoke test that Wails bindings, SQLite, and ent all work together
       - Show connection status in the UI:
         - "Connecting..." while handshake is in progress
         - "Connected - {N} flows" after successful backend communication
         - "Connection failed" if handshake times out or FlowService call fails

    3. Handle the case where Wails-generated types might not exist yet during build:
       - The Wails bindings in `frontend/wailsjs/go/` are generated during `wails dev` or `wails generate module`
       - If imports fail, run `wails generate module` first
       - The service wrapper should handle this gracefully

    Important:
    - The Wails-generated bindings path follows the pattern: `wailsjs/go/{package}/{StructName}`
    - Wails generates model types in `wailsjs/go/models.ts` - use these if available
    - Keep the service wrapper thin - it's a typed facade, not a business logic layer
    - The App.tsx integration is a verification tool, not the final UI (Phase 2 will replace the main content)
