<task type="auto">
  <name>Task 2: Custom SQLite driver, ent schemas, and code generation</name>
  <files>
    internal/infra/sqlite/driver.go

  <action>
    1. Create custom SQLite driver at `internal/infra/sqlite/driver.go`:
       - Register a custom SQL driver named "sqlite3_custom" via `init()` function
       - The driver wraps `modernc.org/sqlite` driver
       - On every `Open()` call, execute two PRAGMAs on the connection:
         - `PRAGMA foreign_keys = ON` (enables FK enforcement)
         - `PRAGMA journal_mode = WAL` (enables concurrent reads)
       - Export a `NewEntClient(dbPath string) (*ent.Client, error)` function that:
         - Opens DB with `sql.Open("sqlite3_custom", dbPath)`
         - Sets `MaxOpenConns(1)` (SQLite single-writer constraint)
         - Sets `MaxIdleConns(1)`
         - Sets `ConnMaxLifetime(0)` (connections live forever)
         - Creates ent client with `ent.NewClient(ent.Driver(entsql.OpenDB("sqlite3", db)))`

       Important: The modernc.org/sqlite driver's `Open()` returns `driver.Conn` which implements `driver.ExecerContext`. Use that interface to execute PRAGMAs. If the interface assertion is problematic, use an alternative approach: open the `*sql.DB` first, then execute PRAGMAs via `db.Exec()` after opening (simpler, equally effective for single-connection pool).

    2. Initialize ent in the project:
       ```
       go run -mod=mod entgo.io/ent/cmd/ent new Flow Node Edge
       ```
       This creates initial schema files in `ent/schema/`. Move or create the schemas in `internal/infra/persistence/ent/schema/` instead. Update the `ent/generate.go` file to point to the correct schema path using `--schemapath` flag.

       Actually, the standard ent approach is to keep schemas in `ent/schema/`. For simplicity and ent compatibility, keep the ent schemas in `ent/schema/` (the standard location) and let ent generate code into `ent/`. This avoids fighting ent's conventions. The Clean Architecture boundary is maintained by the repository implementations in `internal/infra/persistence/repository/` which translate between ent types and domain types.

    3. Define ent schemas in `ent/schema/`:

       `flow.go`:
       - Fields: name (string, NotEmpty), description (string, Optional, Default ""), created_at (Time, Immutable, Default time.Now), updated_at (Time, Default time.Now, UpdateDefault time.Now)
       - Edges: has-many "nodes" (to Node), has-many "edges" (to Edge)

       `node.go`:
       - Fields: type (string, NotEmpty), data (JSON map[string]interface{}, Optional), position_x (Float, Default 0), position_y (Float, Default 0), created_at (Time, Immutable, Default time.Now)
       - Edges: belongs-to "flow" (from Flow, Required), has-many "outgoing_edges" (to Edge), has-many "incoming_edges" (to Edge)
       - Note: Use `edge.From("flow", Flow.Type).Ref("nodes").Unique().Required()` for the FK relationship

       `edge.go`:
       - Fields: source_handle (string, Optional, Default ""), target_handle (string, Optional, Default ""), created_at (Time, Immutable, Default time.Now)
       - Edges: belongs-to "flow" (from Flow, Required), belongs-to "source_node" (from Node, field "source_node_id", Required), belongs-to "target_node" (from Node, field "target_node_id", Required)
       - Note: For source_node and target_node, use `edge.From("source_node", Node.Type).Ref("outgoing_edges").Unique().Required().Field("source_node_id")` pattern

    4. Run ent code generation:
       ```
       go generate ./ent
       ```

    5. Create `ent/generate.go` with:
       ```go
       package ent

       //go:generate go run -mod=mod entgo.io/ent/cmd/ent generate ./schema
       ```

    6. Verify the generated code compiles and update `app.go` startup to use `sqlite.NewEntClient()` and run `client.Schema.Create(ctx)` for auto-migration.

    Important:
    - ent uses integer auto-increment IDs by default. Keep this default (simpler than UUIDs for desktop app).
    - The JSON field for node data uses `map[string]interface{}` type
    - Make sure edge.go schema properly creates FK columns (source_node_id, target_node_id, flow_id)
  </action>
