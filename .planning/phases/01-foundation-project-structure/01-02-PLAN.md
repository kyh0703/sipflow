<task type="auto">
  <name>Task 2: Zustand stores, event hooks, and TypeScript types</name>
  <files>

  <action>
    1. Create `types/responses.ts`:
       - Define `ApiError` type: `{ code: string; message: string }`
       - Define generic `ApiResponse<T>` type: `{ success: boolean; data?: T; error?: ApiError }`
       - These must match the Go `Response[T]` struct from the backend handler layer

    2. Create `types/events.ts`:
       - Define event name constants using "domain:action" convention:
         ```typescript
         export const EVENTS = {
           FRONTEND_READY: 'frontend:ready',
           BACKEND_READY: 'backend:ready',
           FLOW_SAVED: 'flow:saved',
           FLOW_LOADED: 'flow:loaded',
           SIMULATION_NODE_STARTED: 'simulation:node-started',
           SIMULATION_NODE_COMPLETED: 'simulation:node-completed',
         } as const;
         ```
       - Define event payload types for each event

    3. Create `hooks/useWailsEvents.ts`:
       - Generic hook: `useWailsEvents<T>(eventName: string, handler: (data: T) => void, deps?: DependencyList)`
       - Uses `useEffect` to call `EventsOn(eventName, handler)` and returns cleanup via the cancel function returned by EventsOn, or fallback to `EventsOff(eventName)` on unmount
       - Import from `../../wailsjs/runtime/runtime` (Wails auto-generated runtime)

    4. Create `services/eventService.ts`:
       - `initializeEventHandshake()`: async function that:
         a. Sets up `EventsOn('backend:ready', resolve)` listener
         b. Emits `EventsEmit('frontend:ready')` to signal backend
         c. Returns a Promise that resolves when backend:ready is received
         d. Has a timeout (5 seconds) to prevent hanging if backend doesn't respond
       - Import from `../../wailsjs/runtime/runtime`

    5. Create `stores/flowStore.ts`:
       - Interface `FlowState` with: nodes (Node[] from reactflow types - use generic array for now since reactflow isn't installed), edges (Edge[] - use generic array), selectedNodeId (string | null)
       - Actions grouped in `actions` object (Zustand best practice from research):
         - addNode, removeNode, updateNodeData
         - addEdge, removeEdge
         - setNodes, setEdges
         - setSelectedNode
       - Use `create<FlowState>()` from zustand
       - NOTE: Use generic types for nodes/edges for now (e.g., `{ id: string; type: string; position: { x: number; y: number }; data: Record<string, unknown> }[]`). React Flow types will be added in Phase 2.

    6. Create `stores/uiStore.ts`:
       - Interface `UIState` with: sidebarOpen (boolean, default true), activePanel (string | null)
       - Actions: toggleSidebar, setActivePanel
       - Use `create<UIState>()` from zustand

    Important:
    - Use Zustand selectors pattern: export the store hook, consumers use `useFlowStore(state => state.nodes)` not `useFlowStore()`
    - Actions object pattern keeps action references stable (never cause re-renders)
    - useWailsEvents MUST clean up on unmount to prevent memory leaks
    - Do NOT import reactflow types yet (Phase 2) - use plain TypeScript interfaces
  </action>

    - Grep for `create<` in store files confirms Zustand usage
  </verify>
