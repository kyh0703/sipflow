Add a SaveFlow method to FlowService that accepts xyflow canvas state from the frontend and persists it atomically.

Define input types (in flow_service.go or a separate types file if preferred):

```go
// FlowNodeData represents a single xyflow node for persistence
type FlowNodeData struct {
    ID        string                 `json:"id"`
    Type      string                 `json:"type"`
    PositionX float64                `json:"positionX"`
    PositionY float64                `json:"positionY"`
    Data      map[string]interface{} `json:"data"`
}

// FlowEdgeData represents a single xyflow edge for persistence
type FlowEdgeData struct {
    ID           string                 `json:"id"`
    Source       string                 `json:"source"`
    Target       string                 `json:"target"`
    SourceHandle string                 `json:"sourceHandle"`
    TargetHandle string                 `json:"targetHandle"`
    Type         string                 `json:"type"`
    Data         map[string]interface{} `json:"data"`
}

// SaveFlowRequest contains complete xyflow canvas state
type SaveFlowRequest struct {
    FlowID       int            `json:"flowId"`       // 0 for new flow
    Name         string         `json:"name"`
    Nodes        []FlowNodeData `json:"nodes"`
    Edges        []FlowEdgeData `json:"edges"`
    ViewportX    float64        `json:"viewportX"`
    ViewportY    float64        `json:"viewportY"`
    ViewportZoom float64        `json:"viewportZoom"`
}
```

Implement `SaveFlow(req SaveFlowRequest) Response[int]` (returns flow ID):

1. Validate: name must not be empty, entClient must not be nil (return error if no project open).
2. Use ent WithTx pattern for atomic save:
   ```go
   ctx := context.Background()
   tx, err := s.entClient.Tx(ctx)
   ```
3. If FlowID > 0 (update existing):
   - Update flow name, viewport fields, updated_at
   - Delete ALL existing nodes and edges for this flow (cascade via FK or manual delete edges first, then nodes)
   - Recreate nodes and edges from request data
   Why delete-and-recreate: Simpler than diffing. xyflow node IDs are client-generated strings, not database IDs. Mapping between them adds complexity for no benefit at this scale.
4. If FlowID == 0 (new flow):
   - Create flow with name and viewport
   - Create all nodes
   - Create all edges with source/target node references
5. For nodes: store xyflow string ID in a new `xyflow_id` field (need to add this to Node schema). Map xyflow_id to ent node ID for edge references.
   WAIT -- the Node schema doesn't have an xyflow_id field. Add `field.String("xyflow_id").Optional().Default("")` to Node schema. This stores the frontend-generated ID like "sipInstance-1738..." so we can reconstruct the exact state on load.
6. For edges: need xyflow_id too on Edge schema. Add `field.String("xyflow_id").Optional().Default("")`. Also need to store edge type and data JSON.
   Add to Edge schema: `field.String("xyflow_id")`, `field.String("type").Optional().Default("")`, `field.JSON("data", map[string]interface{}{}).Optional()`.
7. Build a map of xyflow_id -> ent node ID during node creation. Use this map when creating edges to set source_node and target_node foreign keys.
8. On success: `tx.Commit()`, return flow ID.
9. On any error: `tx.Rollback()`, return error.

IMPORTANT: Run `go generate ./ent` after schema changes before implementing SaveFlow.

Also update existing FlowService to guard against nil entClient:
```go

    return Failure[...]("NO_PROJECT", "No project is open")
}
```

```go
type FlowState struct {

    ViewportX    float64        `json:"viewportX"`

// FlowMeta represents flow list item (minimal metadata)
type FlowMeta struct {
    ID        int       `json:"id"`
    Name      string    `json:"name"`
    CreatedAt time.Time `json:"createdAt"`
    UpdatedAt time.Time `json:"updatedAt"`
}
```

Implement methods:

1. `LoadFlow(id int) Response[*FlowState]`:
   - Query flow by ID with `.WithNodes()` and `.WithEdges(func(q *ent.EdgeQuery) { q.WithSourceNode().WithTargetNode() })`
   - For edges, we need source and target xyflow_ids. Query edges with their source_node and target_node loaded so we can read xyflow_id from each.
   - Map ent nodes to FlowNodeData: use xyflow_id as ID, type, position_x/y, data JSON.
   - Map ent edges to FlowEdgeData: use xyflow_id as ID, source node's xyflow_id as source, target node's xyflow_id as target, source_handle, target_handle, type, data.
   - Include viewport_x, viewport_y, viewport_zoom from flow.
   - Return complete FlowState.

   - Change return type from `[]*ent.Flow` to `[]FlowMeta` for clean frontend consumption.
   - Query all flows ordered by updated_at desc, map to FlowMeta (id, name, createdAt, updatedAt).

3. `UpdateFlowName(id int, name string) Response[bool]`:
   - Simple update of flow name field.
   - Validate name is not empty.

4. Keep `DeleteFlow(id int) Response[bool]` mostly as-is but ensure it handles cascade delete (ent should cascade delete nodes and edges via FK constraints, but verify with the schema edges configuration).
