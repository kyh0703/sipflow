1. Update `frontend/src/App.tsx`:
   - Listen for Wails events from backend:
     - `project:opened` -> set projectStore.projectPath, call projectStore.actions.refreshFlowList(), clear canvas (setNodes([]), setEdges([])), set currentFlowId to null.
     - `project:created` -> same as opened but flow list will be empty.
     - `project:closed` -> clear projectPath, flows, currentFlowId, clear canvas.
     - `menu:save` -> trigger save logic (see below).
   - Use `EventsOn` from `@wailsio/runtime` or `wailsjs/runtime` for event listeners. Clean up with `EventsOff` in useEffect cleanup.
   - Save logic (triggered by menu:save event):
     - Get current flow state from flowStore (nodes, edges).
     - Get viewport from xyflow instance using `useReactFlow().getViewport()` -- but this hook only works inside ReactFlowProvider. Since App.tsx is inside ReactFlowProvider, use the hook there OR create a save function that FlowCanvas exposes.
     - Alternative: Store viewport in flowStore when it changes (onMoveEnd callback in FlowCanvas). Then save reads from store.
     - Call flowService.saveFlow with flowId (from projectStore.currentFlowId, 0 if new), name, nodes, edges, viewport.
     - On success: update projectStore.currentFlowId with returned flow ID, call markClean(), refreshFlowList() to update timestamps.
     - If no project is open, show alert "No project open. Use File > New or File > Open first."
   - Remove the existing FlowService smoke test code from useEffect (the `flowService.listFlows()` test call). It was from Phase 1 verification and is no longer needed.

2. Update `frontend/src/components/flow/FlowCanvas.tsx`:
   - Add `onMoveEnd` handler to ReactFlow that stores viewport in flowStore:
     ```tsx
     const onMoveEnd = useCallback((_event: any, viewport: Viewport) => {
       // Store viewport for save operations
       flowStore.actions.setViewport(viewport)
     }, [])
     ```
   - Add viewport state to flowStore (see below).

3. Update `frontend/src/stores/flowStore.ts`:
   - Add viewport state: `viewport: { x: 0, y: 0, zoom: 1 }`.
   - Add `setViewport` action.
   - Add dirty state subscription: After any node/edge/data change (onNodesChange, onEdgesChange, onConnect, addNode, removeNode, updateNodeData, addEdge, removeEdge), call `useProjectStore.getState().actions.markDirty()`.
   - Import useProjectStore for cross-store communication. Use `getState()` pattern (not hook) since this is inside a non-component context:
     ```typescript
     import { useProjectStore } from './projectStore'
     // Inside an action:
     useProjectStore.getState().actions.markDirty()
     ```

4. Update `frontend/src/components/layout/Header.tsx`:
   - Show project name from projectStore.projectPath (extract filename from path, remove .sipflow extension).
   - Show dirty indicator: if isDirty, append " *" to title or show a dot indicator.
   - If no project open, show "SIPFlow" or "No Project" as title.
   - Format: "ProjectName - FlowName *" when dirty, "ProjectName - FlowName" when clean.
   - Read currentFlowId from projectStore and find flow name from flows array.

5. Handle loading a flow when user clicks in sidebar:
   - The LeftSidebar click handler should:
     a. Call flowService.loadFlow(flowId) to get FlowState from backend.
     b. Call flowStore.actions.setNodes(flowState.nodes) -- but nodes need to be converted from FlowNodeData to xyflow Node format (add `position: {x, y}` wrapper).
     c. Call flowStore.actions.setEdges(flowState.edges) -- edges need xyflow Edge format.
     d. Set viewport via `useReactFlow().setViewport({x, y, zoom})` -- this needs to be in a component inside ReactFlowProvider. Use FlowCanvas or a custom hook.
     e. Set projectStore.currentFlowId.
     f. Call projectStore.actions.markClean() since we just loaded fresh state.
   - Create a helper function or hook `useFlowPersistence()` in a new file or in App.tsx that handles the save/load/conversion logic between frontend xyflow format and backend FlowState format.

   Conversion from backend FlowState to xyflow:
   ```typescript
   // Backend FlowNodeData -> xyflow Node
   const nodes: Node[] = flowState.nodes.map(n => ({
     id: n.id,
     type: n.type,
     position: { x: n.positionX, y: n.positionY },
     data: n.data,
   }))

   // Backend FlowEdgeData -> xyflow Edge
   const edges: Edge[] = flowState.edges.map(e => ({
     id: e.id,
     source: e.source,
     target: e.target,
     sourceHandle: e.sourceHandle || undefined,
     targetHandle: e.targetHandle || undefined,
     type: e.type || 'flowEdge',
     data: e.data,
   }))
   ```

   Conversion from xyflow to backend (for save):
   ```typescript
   // xyflow Node -> backend FlowNodeData
   const nodes = flowStore.nodes.map(n => ({
     id: n.id,
     type: n.type || '',
     positionX: n.position.x,
     positionY: n.position.y,
     data: n.data,
   }))
   ```
