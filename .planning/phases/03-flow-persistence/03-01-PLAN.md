<objective>
Establish project-based .sipflow file infrastructure with runtime SQLite database switching and Wails native menu. All ent schema updates for Phase 3 are consolidated here so downstream plans start with complete schemas.

Purpose: Phase 3 transitions from a single app-internal database to per-project .sipflow files. Each project is an independent SQLite database that users can share, copy, and version control. This plan creates the Go backend infrastructure for opening/closing project files and the Wails File menu.

Design decision: Viewport persistence (zoom/pan position) is included as a quality-of-life feature. The CONTEXT.md lists this under "Claude's Discretion" and we choose to persist it because restoring exact canvas position on reload significantly improves UX.

Output: All ent schema updates (Flow viewport, Node xyflow_id, Edge xyflow_id/type/data), ProjectService with runtime DB switching, Wails native File menu (New/Open/Save As).
</objective>

  <action>
1. Update `ent/schema/flow.go` to add viewport fields for canvas state persistence:
   - `viewport_x` float64, default 0
   - `viewport_y` float64, default 0
   - `viewport_zoom` float64, default 1
   These store the xyflow viewport so users see exact canvas position on reload.

2. Update `ent/schema/node.go` to add xyflow frontend ID:
   - `field.String("xyflow_id").Optional().Default("")` - stores the frontend-generated ID like "sipInstance-1738..." so we can reconstruct exact xyflow state on load.
   This is needed because ent uses auto-increment integer IDs but xyflow uses string IDs generated on the frontend. Without this field, loaded flows cannot restore correct edge connections.

3. Update `ent/schema/edge.go` to add xyflow frontend ID and edge metadata:
   - `field.String("xyflow_id").Optional().Default("")` - stores frontend-generated edge ID
   - `field.String("type").Optional().Default("")` - stores xyflow edge type (e.g., "flowEdge")
   - `field.JSON("data", map[string]interface{}{}).Optional()` - stores edge data (validation state, custom properties)
   These fields are needed for full xyflow state reconstruction. Without edge type/data, validation visual state would be lost on reload.

4. Run `go generate ./ent` to regenerate ent code after all schema changes.

5. Create `internal/handler/project_service.go` with ProjectService struct:
   ```go
   type ProjectService struct {
       ctx           context.Context
       entClient     *ent.Client
       currentPath   string
       flowService   *FlowService
   }
   ```

   Methods:
   - `NewProjectService(flowService *FlowService) *ProjectService`
   - `SetContext(ctx context.Context)` - called from app.startup
   - `NewProject() Response[bool]` - Show Wails SaveFileDialog with .sipflow filter, create new DB file at chosen path, run schema migration, close previous client if any, set currentPath. Emit `project:created` event. Return error if user cancels dialog.
   - `OpenProject() Response[string]` - Show Wails OpenFileDialog with .sipflow filter, open DB file, run schema migration (Schema.Create - idempotent), close previous client if any, update FlowService's entClient via SetEntClient, set currentPath. Emit `project:opened` event. Return the path on success.
   - `CloseProject() Response[bool]` - Close current ent client, clear currentPath, set FlowService entClient to nil. Emit `project:closed` event.
   - `SaveProjectAs() Response[string]` - Show SaveFileDialog, copy current DB file to new path (use io.Copy), reopen at new path. Emit `project:opened` event with new path. Return new path. If no project open, return error.
   - `CurrentProjectPath() string` - Return currentPath (empty string if no project open).
   - `IsProjectOpen() bool` - Return currentPath != "".
   - `openDatabase(path string) error` - Private helper: close existing client, call sqlite.OpenEntDriver(path), create ent.NewClient, run Schema.Create, update flowService.SetEntClient. This is the core DB switching logic used by NewProject, OpenProject, SaveProjectAs.

   Use Wails runtime for dialogs:
   ```go
   import "github.com/wailsapp/wails/v2/pkg/runtime"

   path, err := runtime.SaveFileDialog(s.ctx, runtime.SaveDialogOptions{
       Title: "Create New Project",
       DefaultFilename: "untitled.sipflow",
       Filters: []runtime.FileFilter{
           {DisplayName: "SIPFlow Project", Pattern: "*.sipflow"},
       },
   })
   ```

   IMPORTANT: All project lifecycle events (project:created, project:opened, project:closed) are emitted from ProjectService methods, NOT from main.go menu callbacks. This keeps event emission consistent -- ProjectService is the single source of truth for project state transitions. Menu callbacks call ProjectService methods, which handle both the operation AND the event emission.

   IMPORTANT: Always close previous ent client before opening new one to prevent "database is locked" errors. Use a mutex or sequential access pattern since Wails calls are already single-threaded from frontend.

6. Update `internal/infra/sqlite/driver.go` - no changes needed, existing OpenEntDriver works for any path. Just verify it handles paths with spaces correctly (it does, since it uses fmt.Sprintf with file: prefix).

   - In startup(), call `a.projectService.SetContext(ctx)` instead of the current single-DB initialization. REMOVE the existing database initialization code (configDir, dbPath, sqlite.OpenEntDriver, ent.NewClient, Schema.Create, flowService.SetEntClient). The app now starts with NO database open.

2. Update `main.go`:
   - Add Wails native menu using `menu.NewMenu()`:
     ```go
     import "github.com/wailsapp/wails/v2/pkg/menu"
     import "github.com/wailsapp/wails/v2/pkg/menu/keys"

     appMenu := menu.NewMenu()
     fileMenu := appMenu.AddSubmenu("File")
     fileMenu.AddText("New Project", keys.CmdOrCtrl("n"), func(cd *menu.CallbackData) {
         app.projectService.NewProject()
     })
     fileMenu.AddText("Open Project...", keys.CmdOrCtrl("o"), func(cd *menu.CallbackData) {
         app.projectService.OpenProject()
     })
     fileMenu.AddSeparator()
     fileMenu.AddText("Save", keys.CmdOrCtrl("s"), func(cd *menu.CallbackData) {
         // Save needs frontend canvas state, so emit event for frontend to handle
         wailsRuntime.EventsEmit(app.ctx, "menu:save")
     })
     fileMenu.AddText("Save As...", keys.CmdOrCtrl("shift+s"), func(cd *menu.CallbackData) {
         app.projectService.SaveProjectAs()
     })
     ```
   - Add `Menu: appMenu` to wails.Run options.
   - Add `app.projectService` to the Bind slice.

   IMPORTANT: The "Save" menu item is the ONLY event emitted from main.go. This is intentionally different from project lifecycle events because Save requires current xyflow canvas state which lives in the frontend. The frontend listens for "menu:save", gathers canvas state, and calls backend SaveFlow. All other project events (project:opened, project:created, project:closed) come from ProjectService methods which the menu callbacks invoke.
