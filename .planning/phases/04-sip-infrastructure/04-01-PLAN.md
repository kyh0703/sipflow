<task type="auto">
  <name>Task 2: Create SIPService handler with CRUD and wire into Wails app</name>
  <files>
    internal/handler/sip_service.go
    app.go
    main.go
  </files>
  <action>
    1. Create `internal/handler/sip_service.go` following FlowService pattern:

       ```go
       type SIPService struct {
           entClient *ent.Client
           emitter   *EventEmitter
       }
       ```

       - `NewSIPService(emitter *EventEmitter) *SIPService` -- entClient starts nil (same as FlowService)
       - `setEntClient(client *ent.Client)` -- unexported to prevent Wails binding (same pattern as FlowService)

       Define request/response types:
       - `SIPServerMeta` struct: ID (int), Name, Address (string), Port (int), Transport (string) -- no password in list response
       - `CreateServerRequest` struct: Name, Address, Port, Transport, Username, Password
       - `UpdateServerRequest` struct: same fields as Create

       CRUD methods (all guard nil entClient with NO_PROJECT error, same as FlowService):
       - `ListServers() Response[[]SIPServerMeta]` -- query all, ordered by name, map to SIPServerMeta (exclude password)
       - `CreateServer(req CreateServerRequest) Response[int]` -- validate name+address NotEmpty, create in ent, return ID
       - `UpdateServer(id int, req UpdateServerRequest) Response[bool]` -- validate, update in ent
       - `DeleteServer(id int) Response[bool]` -- delete by ID
       - `GetServer(id int) Response[SIPServerMeta]` -- get single server by ID (for editing)

    2. Update `app.go`:
       - Add `sipService *handler.SIPService` field to App struct
       - In `NewApp()`: create SIPService with eventEmitter
       - In `startup()`: no extra setup needed (entClient set via ProjectService)

    3. Update `main.go`:
       - Add `app.sipService` to the Wails `Bind` slice

    4. Update `internal/handler/project_service.go`:
       - Add `sipService *SIPService` field
       - Update `NewProjectService` to accept sipService parameter
       - In `openDatabase()`: call `s.sipService.setEntClient(client)` alongside flowService
       - In `CloseProject()`: call `s.sipService.setEntClient(nil)` alongside flowService

    5. Update `app.go` NewApp to pass sipService to NewProjectService:
       ```go
       sipService := handler.NewSIPService(eventEmitter)
       projectService := handler.NewProjectService(flowService, sipService)
       ```

    6. Verify: `go build ./...`
  </action>

    - `app.go` has sipService field and creates it in NewApp

    - `project_service.go` passes entClient to sipService on openDatabase/CloseProject
  </verify>
