## 요구사항 결과

### Transfer (통화 전환)

- [x] **XFER-01**: 사용자가 BlindTransfer Command 노드를 캔버스에 배치하여 활성 통화를 제3자 URI로 전환(REFER)할 수 있음
  - **결과:** 검증됨 — executeBlindTransfer → referrer.Refer() + Hangup() 파이프라인 구현
  - **구현:** Phase 11, Plans 11-01/11-02

- [x] **XFER-02**: 사용자가 TransferEvent 노드로 상대방이 보낸 REFER 요청을 감지하고 대기할 수 있음
  - **결과:** 검증됨 — OnRefer 콜백 → Refer-To URI 추출 + Invite/Ack + TRANSFERRED 이벤트
  - **구현:** Phase 11, Plans 11-01/11-02

### Hold/Retrieve (통화 보류/해제)

- [x] **HOLD-01**: 사용자가 Hold Command 노드를 배치하여 활성 통화를 보류(Re-INVITE sendonly)할 수 있음
  - **결과:** 검증됨 — executeHold → MediaSession.Mode(sdp.ModeSendonly) + ReInvite()
  - **구현:** Phase 10, Plans 10-01/10-02

- [x] **HOLD-02**: 사용자가 Retrieve Command 노드를 배치하여 보류된 통화를 재개(Re-INVITE sendrecv)할 수 있음
  - **결과:** 검증됨 — executeRetrieve → MediaSession.Mode(sdp.ModeSendrecv) + ReInvite()
  - **구현:** Phase 10, Plans 10-01/10-02

- [x] **HOLD-03**: 사용자가 HeldEvent 노드로 상대방의 Hold Re-INVITE를 감지할 수 있음
  - **결과:** 검증됨 — OnMediaUpdate → LocalSDP recvonly 감지 → emitSIPEvent("HELD")
  - **구현:** Phase 10, Plans 10-01/10-02

- [x] **HOLD-04**: 사용자가 RetrievedEvent 노드로 상대방의 Retrieve Re-INVITE를 감지할 수 있음
  - **결과:** 검증됨 — OnMediaUpdate → LocalSDP sendrecv 감지 → emitSIPEvent("RETRIEVED")
  - **구현:** Phase 10, Plans 10-01/10-02

### UI 개선 (레이아웃 리디자인)

- [x] **UI-01**: 좌측에 아이콘 네비게이션 바(Activity Bar)를 두고, 아이콘 클릭 시 해당 사이드바 패널(시나리오 트리, 노드 팔레트)이 토글됨
  - **결과:** 검증됨 — ActivityBar 컴포넌트 + handlePanelToggle 상태 관리
  - **구현:** Phase 12, Plan 12-01

- [x] **UI-02**: 사이드바가 shadcn Resizable 컴포넌트로 너비 조절 가능함
  - **결과:** 검증됨 — ResizablePanelGroup + panelRef imperative handle + collapsible
  - **구현:** Phase 12, Plan 12-01

- [x] **UI-03**: 새 Command/Event 노드(Hold, Retrieve, BlindTransfer, TransferEvent, HeldEvent, RetrievedEvent)에 맞는 Properties 패널, 아이콘, 팔레트 항목이 추가됨
  - **결과:** 검증됨 — COMMAND_ICONS 매핑 + node-palette 확장 + command/event-properties 분기
  - **구현:** Phase 13, Plan 13-01

### 비기능 요구사항

- [x] **NF-01**: 새 Command/Event 핸들러에 대한 Go 단위 테스트가 포함됨
  - **결과:** 검증됨 — Hold/Retrieve/BlindTransfer/TRANSFERRED 에러 경로 + 이벤트 버스 테스트
  - **구현:** Phase 10/11/13, Plans 10-02/11-02/13-02

- [x] **NF-02**: 기존 v1.1 시나리오가 깨지지 않음 (하위 호환성 유지)
  - **결과:** 검증됨 — TestParseScenario_V1_1_BackwardCompatibility PASS
  - **구현:** Phase 13, Plan 13-02

- [x] **NF-03**: 새 노드가 기존 노드 팔레트의 드래그앤드롭 패턴과 일관되게 동작함
  - **결과:** 검증됨 — Command=blue, Event=amber 동일 패턴, 사용자 시각적 검증 통과
  - **구현:** Phase 13, Plan 13-01

## 범위 밖 (이유와 함께)

| 기능 | 제외 이유 |
|------|-----------|
| Attended Transfer | 복잡도 매우 높음 (SessionStore 1:N 리팩토링 + diago Replaces 미지원) |
| Hold Music 자동 재생 | 사용자가 PlayAudio 노드로 직접 구성 |
| SIP INFO 기반 Hold | Re-INVITE 방식만 지원 (RFC 6337 표준) |
| 실시간 미디어 모니터링 | RTP 스트림 시각화는 범위 밖 |
